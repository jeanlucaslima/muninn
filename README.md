# Muninn

> A local-first memory daemon.

Muninn records what you copy and turns it into something you can **search, recall, and reuse**.

Named after Muninn, Odin’s raven that brings back everything it has seen.

---

## What it is

Muninn is not just a clipboard manager.

It is a **memory system**:

* continuously observes your clipboard
* stores history locally
* makes it queryable through a CLI and UI

Clipboard is the first input. Not the limit.

---

## Components

### `muninnd` — daemon

* runs in the background (launchd)
* watches the clipboard (`NSPasteboard`)
* normalizes and deduplicates content
* persists history (SQLite)
* exposes a local interface

---

### `muninn` — CLI

Your primary interface.

```bash
muninn list
muninn search "stripe webhook"
muninn latest
muninn get 42
muninn copy 42
```

Structured output:

```bash
muninn list --json
```

---

### Menu Bar

* browse recent entries
* search instantly
* restore or paste
* pin important items
* pause/resume capture

Default shortcut: `Cmd + Option + V`

---

## Architecture

```text
Clipboard → muninnd → SQLite
                     ↓
          ┌──────────┴──────────┐
          │                     │
       CLI (`muninn`)   Menu Bar
```

Principles:

* daemon is the source of truth
* clients do not access storage directly
* everything is local

---

## Why

The clipboard is a stream of high-signal data:

* code snippets
* links
* commands
* ideas

Muninn treats it as **memory**, not a buffer.

---
